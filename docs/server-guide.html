Server Setup

Guide to configuring and connecting the Claude Wheel server

What is the server

The server is the link between the mobile app and Claude Code. It receives voice requests from the phone, sends text to the tmux session running Claude, waits for the response, and returns it back.

The server runs on the machine where Claude Code CLI is installed — this can be your computer, a home server, or a VPS.

Requirements

  • Ubuntu 22+ (or another Linux)
  • Python 3.10+
  • tmux, ffmpeg
  • Claude Code CLI
  • A Groq account for STT (speech recognition)

Installation

System dependencies

sudo apt install ffmpeg tmux

Clone the repository

git clone https://github.com/dim4ai/claude-wheel.git
cd claude-wheel

Virtual environment

cd claude-wheel-server
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Environment variables

Create the .env file:

cp .env.example .env
API_KEY — a secret key that is also used in the mobile app in the Server API Key field. Minimum 20 characters, must contain digits, lowercase and uppercase letters — the server will not start if the key is too simple.
Groq API key is entered in the mobile app and sent to the server with each request. Alternatively, you can set GROQ_API_KEY in .env, in which case the key in the mobile app is not needed.

Session list

The session list is stored in the file specified in .env (variable SESSIONS_LIST):

agent:/home/user/projects/agent
coach-lab:/home/user/projects/coach/lab
coach-discussion:/home/user/projects/coach/discussion

Each line defines a tmux session name and project directory separated by a colon. Switching between sessions is available from the mobile app.

Auto-start

Create a systemd service for auto-start:

./setup-service.sh

The service will start automatically on reboot.

stop hook — capturing Claude responses

For the server to receive Claude responses, you need to configure the stop hook in Claude Code:

./setup-hook.sh

Verification

After starting, make sure the server is running:

curl http://localhost:{PORT}/health
# Expected response: {"status":"ok","stt":"...","tts":"edge-tts"}

Phone access

The phone must have network access to the server. Choose the appropriate option:

Option 1 — Home Wi-Fi

Find the computer's IP address on the local network:

ip addr

In the mobile app, enter http://192.168.x.x:{PORT}.

Option 2 — Public IP + domain + trusted certificate

If you have a server with a public IP, you will need a domain name and a trusted certificate. Both can be obtained for free via DuckDNS (domain) and Let's Encrypt (certificate via Certbot with the certbot-dns-duckdns plugin).

Since HTTPS runs on the standard port 443 and the Claude Wheel server listens on your chosen {PORT}, Nginx is needed between them as a reverse proxy. If there is already something on the same domain, it is convenient to place Claude Wheel under a prefix — for example /agent. If the domain is used only for Claude Wheel, no prefix is needed and in the mobile app you can simply enter https://your-domain.duckdns.org.

# Example Nginx config with /agent prefix
server {
    listen 443 ssl;
    server_name your-domain.duckdns.org;

    ssl_certificate /etc/letsencrypt/live/your-domain.duckdns.org/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.duckdns.org/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    location /agent/ {
        proxy_pass http://127.0.0.1:{PORT}/;
        proxy_set_header Host $host;
    }
}

In the mobile app, enter https://your-domain.duckdns.org/agent.

Traffic between the phone and the server is encrypted — this is important if the server is accessible from the internet.

Option 3 — Public IP + self-signed certificate

Generate a certificate with the server's IP address in the SAN field, install it on the phone as a trusted CA, and connect directly via IP without a domain.

openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem -days 365 -nodes \
  -subj "/CN=Claude Wheel" \
  -addext "subjectAltName=IP:x.x.x.x"

Copy the certificate and key to the Nginx directory:

sudo cp cert.pem /etc/nginx/cert.pem
sudo cp key.pem /etc/nginx/key.pem
# Example Nginx config listening on port 4000
server {
    listen 4000 ssl;
    server_name _;

    ssl_certificate     /etc/nginx/cert.pem;
    ssl_certificate_key /etc/nginx/key.pem;

    location /agent/ {
        proxy_pass http://127.0.0.1:{PORT}/;
        proxy_set_header Host $host;
    }
}

Install the cert.pem file on the phone as a trusted CA. In the mobile app, enter https://x.x.x.x:4000/agent.

Option 4 — Tailscale + HTTP

Install Tailscale on the computer and phone, sign in under the same account. Traffic goes through an encrypted WireGuard tunnel, so HTTP is safe. In the mobile app, enter http://100.x.x.x:{PORT}.

API endpoints

MethodPathDescription
POST/sttAudio → text (Groq Whisper)
POST/ttsText → MP3 (Edge TTS)
POST/askSend text to Claude, get a response
POST/dispatchSession management (list / switch / create / recreate / close)
GET/screenSnapshot of the current tmux screen
POST/keypressSend a key to the session (Up / Down / Enter / Escape)
GET/healthServer availability check
All endpoints (except /health) require the x-api-key header or the ?api_key= parameter.
